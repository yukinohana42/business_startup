# 05_ACCEPTANCE_CRITERIA

## 1. 全体合格条件

このプロジェクトの成果物は、次の条件を満たしたときに合格とする。

- 全日程が4日間で統一されている。
- 起業を押し付けていない。
- 不安を煽っていない。
- 中学2年生が読んで意味がわかる。
- 親が読んでも安心できる。
- 価値を「相手の変化」として扱っている。
- お金を「価値提供の対価」として扱っている。
- PMFを「作ったものと欲しい人の気持ちがピタッとはまること」と言い換えている。
- 小さく試し、反応から学ぶ流れがある。
- 子供が楽しそうだと思えるミッション構造になっている。
- 親が正解を教える構成ではなく、問いを出す構成になっている。

## 2. 子供向け教材の合格条件

- 短い問いから始まる。
- 身近な例がある。
- 書く、選ぶ、話す、試すの行動がある。
- 1ページまたは1スライドに情報を詰め込みすぎていない。
- 難しい言葉には言い換えがある。
- 仕事、お金、価値の話が抽象論で終わっていない。
- 「誰がどう変わるか」を書く欄がある。
- 失敗ではなく赤信号として扱っている。

## 3. ワークシートの合格条件

- A4印刷で使える余白がある。
- 記入欄が広すぎず、狭すぎない。
- 1つの欄に長文を書かせない。
- わからない欄があっても進められる。
- 最後に次の行動がある。
- Day1からDay4まで学びが積み上がる。

## 4. 親向け資料の合格条件

- 親を先生、採点者、ゲームマスターとして扱っていない。
- 親の役割が、問いを出す伴走者として説明されている。
- OK/NG声かけが具体的である。
- 子供の結果ではなく、観察、問い、小さな挑戦、改善を支える内容になっている。
- 不安を煽る表現がない。

## 5. 子供向けスライドの合格条件

- 各日8〜12枚程度。
- 1スライド1メッセージ。
- 本文は3〜5行まで。
- ワーク指示が明確。
- Zoomで読める。
- スライドだけで完結させすぎず、話す補足は台本側に置ける。

## 6. インフォグラフィックpromptの合格条件

- 1ページ1メッセージ。
- 4〜6枚程度に分けている。
- 価値をBefore/Afterで見せている。
- 人物イラストが意味を持っている。
- 既存作品、既存キャラクター、特定作家の絵柄模倣がない。
- 文字量を抑えている。
- 日本語本文は後編集可能な設計になっている。

## 7. 禁止表現チェック

次の表現が含まれていたら修正する。

- 起業しないといけない。
- 会社員は危ない。
- サラリーマンは終わり。
- 将来困るから今やりなさい。
- 正解はこれ。
- それは無理。
- もっとすごいアイデアにしなさい。
- 稼げる人が偉い。
- 点数や順位で比べる。

## 8. 完成前チェックリスト

- [ ] 4日間構成になっている。
- [ ] 旧構成の記述が残っていない。
- [ ] `SOURCE_OF_TRUTH.md` と矛盾していない。
- [ ] 子供向けと親向けのトーンが混ざっていない。
- [ ] すべての成果物が同じ用語で統一されている。
- [ ] ワークシートに書く欄がある。
- [ ] 親向け資料にOK/NG声かけがある。
- [ ] 子供向けスライドに身近な例がある。
- [ ] インフォグラフィックpromptが既存IP模倣を避けている。
- [ ] docs-only作業にPDF、PPTX、PNG、ブラウザプロファイルが混ざっていない。
- [ ] `.env`、Cookie、Login Data、Cache、Crashpad dump、秘密情報が含まれていない。
