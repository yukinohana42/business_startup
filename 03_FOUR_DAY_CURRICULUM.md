# 03_FOUR_DAY_CURRICULUM

## 全体設計

4日間で、子供たちが「価値を見つける → 具体的な誰かに向ける → 小さく試す → 価値の変化を伝える」流れを体験する。

各日は45分を標準とする。30分版と60分版にも調整できるようにする。

## 4日間の学習ストーリー

| Day | ミッション | 子供の状態変化 | 中心ワーク | 最終アウトプット |
|---|---|---|---|---|
| Day1 | お金と仕事の正体を見破れ | 仕事を「お金をもらうこと」だけでなく「誰かが助かること」と見られる | 価値変換ワーク | 仕事とは、＿＿を＿＿に変えること |
| Day2 | 日常に隠れた価値の種を見つけろ | 自分の生活の不満、面倒、好きから価値の種を見つけられる | 価値の種ハンティング | 価値の種リスト3つ |
| Day3 | 誰の何を助けるかを形にし、小さく試せ | 「みんな向け」ではなく具体的な誰かに向けて考えられる | アイデア文＋欲しいサイン＋小さな実験 | 私たちは、＿＿の＿＿を＿＿に変える。最初に試すことは＿＿ |
| Day4 | 3分で価値の変化を伝えろ | 自分たちのアイデアを、問題、相手、解決、試し方で伝えられる | 3分ピッチ作成 | 3分ピッチ＋振り返り |

## Day1: Value Detective / 価値探偵

### テーマ

仕事・お金・価値の正体を見つける。

### ゴール

子供が次を言える。

- 仕事は、誰かの状態を少し良くすること。
- 価値は、相手の変化である。
- お金は、価値が届いたことへの対価である。

### 導入問い

「仕事って、何をしたら仕事になると思う？」

### 伝える概念

- 仕事 = 誰かの困った、面倒、退屈を変えること。
- 価値 = 相手のBefore/After。
- お金 = 価値が届いたサイン。

### 身近な例

- ラーメン屋: お腹がすいた → おいしく食べられた。
- 塾: わからない → できそう。
- ゲーム攻略動画: クリアできない → 進めた。
- コンビニ: 今すぐ欲しい → すぐ買えた。

### メインワーク

価値変換ワーク。

1. 身近な仕事を1つ選ぶ。
2. その仕事で助かる人を書く。
3. Beforeを一言で書く。
4. Afterを一言で書く。
5. 「仕事とは、＿＿を＿＿に変えること」と書く。

### 45分進行

| 時間 | 内容 |
|---|---|
| 0〜5分 | 今日のミッション説明 |
| 5〜12分 | 導入問い「仕事って何？」 |
| 12〜20分 | 価値 = 相手の変化の説明 |
| 20〜32分 | 価値変換ワーク |
| 32〜40分 | 2人で共有 |
| 40〜45分 | Final Missionと宿題 |

### 宿題

家の中か街の中で、誰かが助かっている仕事を1つ見つける。

## Day2: Problem Hunter / 困りごとハンター

### テーマ

価値の種を日常から見つける。

### ゴール

子供が次を言える。

- 価値の種は、不満、面倒、好き、欲しい気持ちの中にある。
- 価値は「大発明」だけではなく、身近な少しの改善から生まれる。

### 導入問い

「今日1日で、ちょっと面倒だったことは何？」

### 伝える概念

- 困った、面倒、退屈、もっと楽しくしたい、もっとやりたい、好き、欲しい。
- 価値の種は、普段の生活に隠れている。
- 困っている人を笑わない。観察する。

### 身近な例

- 部活の準備が面倒。
- テスト前に何から勉強すればいいかわからない。
- ゲームの情報が多すぎて探しづらい。
- 友達と遊ぶ予定を決めるのが面倒。
- 文化祭の係で忘れ物が多い。

### メインワーク

価値の種ハンティング。

1. 自分、友達、家族、学校、街の5つの観察場所から選ぶ。
2. 「困った」「面倒」「好き」「欲しい」を書き出す。
3. 一番気になるものを1つ選ぶ。
4. その人は今どう解決しているかを考える。
5. それが変わると何がうれしいかを書く。

### 45分進行

| 時間 | 内容 |
|---|---|
| 0〜5分 | 前回の観察共有 |
| 5〜12分 | 価値の種の説明 |
| 12〜25分 | 価値の種ハンティング |
| 25〜35分 | 1つ選んで深掘り |
| 35〜42分 | 2人で共有 |
| 42〜45分 | Final Missionと宿題 |

### 宿題

選んだ価値の種について、1人に「今どうしてる？」と聞いてくる。個人情報は聞かない。

## Day3: Idea & Want Signal Builder / アイデア・欲しいサインビルダー

### テーマ

誰の何を解決するかを形にし、本当に欲しい人がいるか小さく試す。

### ゴール

子供が次を言える。

- 助けたい人は「みんな」ではなく、具体的な誰か。
- 良いアイデアは、自分が作りたいものではなく、相手が本当に欲しいものに近いもの。
- 欲しいサインを見ると、作ったものと欲しい人の気持ちが近いかがわかる。
- 最初は小さく試せばよい。

### 導入問い

「そのアイデアを、本当に使いたい人は誰？」

### 伝える概念

- 助けたい人 = 困っている具体的な人。
- アイデア文 = 私たちは、＿＿の＿＿を＿＿に変える。
- 欲しいサイン = 欲しい、使いたい、続けたい、人に言いたい。
- 赤信号 = 失敗ではなく、変えるサイン。

### 身近な例

- 「学生みんな」ではなく「テスト前に何から始めるか迷う中2」。
- 「部活を便利にする」ではなく「朝練前に忘れ物が多いサッカー部員」。
- 「ゲーム情報サイト」ではなく「攻略動画を見る時間が長くなりすぎる人」。

### メインワーク

アイデア文と小さな実験。

1. 具体的な相手を1人決める。
2. その人のBeforeを書く。
3. Afterを書く。
4. 解決案を3つ出す。
5. 一番小さく試せる案を選ぶ。
6. 何を見たら「欲しい」とわかるかを書く。

### 45分進行

| 時間 | 内容 |
|---|---|
| 0〜6分 | 前回の聞いたこと共有 |
| 6〜14分 | 助けたい人と欲しいサインの説明 |
| 14〜28分 | アイデア文ワーク |
| 28〜38分 | 小さな実験設計 |
| 38〜43分 | 赤信号なら何を変えるか |
| 43〜45分 | Final Mission |

### 宿題

できれば1つだけ、相手に聞く、見せる、紙で試す、チャットで聞くなどの小さな実験をする。

## Day4: Pitch Challenger / ピッチチャレンジャー

### テーマ

価値の変化を3分で伝える。

### ゴール

子供が次を言える。

- 誰の何をどう変えるアイデアか。
- なぜその人に必要そうか。
- 最初にどう試すか。
- 反応が悪かったらどこを変えるか。

### 導入問い

「聞いた人が『なるほど、それ誰か助かりそう』と思うには、何を伝えればいい？」

### 伝える概念

ピッチは、すごさ自慢ではない。価値の変化を相手に伝えること。

### 3分ピッチの型

1. 誰が困っているか。
2. 何に困っているか。
3. 今はどうしているか。
4. 自分たちのアイデアでどう変わるか。
5. 最初にどう小さく試すか。
6. 反応が悪かったら何を変えるか。

### メインワーク

3分ピッチ作成と発表。

### 45分進行

| 時間 | 内容 |
|---|---|
| 0〜5分 | 今日のゴール説明 |
| 5〜12分 | ピッチの型説明 |
| 12〜25分 | ピッチ台本作成 |
| 25〜35分 | 1回目発表 |
| 35〜42分 | 聞き手メモと改善 |
| 42〜45分 | 起業セミナーに向けたまとめ |

### 最終アウトプット

- 3分ピッチ。
- 価値の変化カード。
- 次に聞いてみたいこと。
- セミナー本番で注目したいこと。

## 30分版への短縮方法

- 導入説明を5分以内にする。
- 例を1つに絞る。
- ワークは1つだけにする。
- 発表は1人1分にする。
- 宿題を軽くする。

## 60分版への拡張方法

- 例を2つ増やす。
- カードを使う時間を増やす。
- 相互質問を入れる。
- 1回目の発表後に改善タイムを入れる。
- 振り返りを丁寧に行う。
