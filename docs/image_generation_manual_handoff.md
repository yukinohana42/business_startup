# 画像生成 Manual Handoff Guide

## 基本方針

Codexは画像生成を直接実行しない。

Codexは、画像生成用のprompt packetをMarkdownで作る。ユーザーがそのprompt packetをGPT画像生成などへ貼り、候補画像を作る。

## 画像生成で作るもの

- 子供向けインフォグラフィック。
- 親向け説明用インフォグラフィック。
- 価値のBefore/Afterを見せる図解。
- 探偵ノート、カード、実験メモなどの背景イラスト。

## 画像生成で任せすぎないもの

- 正確な日本語本文。
- 長い表。
- OK/NG声かけの細かい文面。
- ワークシートの記入欄。
- 法的、教育的に正確な説明。

これらはMarkdown、HTML、スライド側で後編集する。

## Prompt Packetの必須項目

1. 目的。
2. 画面比率。
3. ページ番号。
4. 1ページのメッセージ。
5. キャラクター行動。
6. Before/After。
7. 図解要素。
8. 色と雰囲気。
9. 入れてよい短い文字。
10. 入れてはいけないもの。
11. 後編集で載せる日本語本文。
12. 保存先。
13. 人間レビュー観点。

## 保存先ルール

候補画像は必ず `candidates/` に置く。

例:

```text
assets/generated/day1/page_01/candidates/day1_page_01_candidate_YYYYMMDD_01.png
```

人間レビュー前に `final/` へ置かない。

## 禁止

- 既存作品のキャラクターを使う。
- 特定アニメ、漫画、ゲームの絵柄を指定する。
- ロゴを模倣する。
- 文字が大量に入った1枚画像にする。
- 子供を不安にさせる演出にする。
- 稼げることを煽る演出にする。
