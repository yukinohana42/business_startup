# 全体構成: 4日間・中学2年生向け「価値を見つける」事前ワークショップ

## 一言説明

中学2年生の子供2人が、起業セミナーに参加する前に、家庭内で4日間のワークショップを行い、「誰かにとって価値あることを見つけ、形にし、小さく試す力」を育てるプロジェクトである。

この資料は、ユーザー、親、Codex、他のAIが同じ理解で作業するための全体地図である。

## この取り組みの目的

起業を押し付けるためではない。

会社に入るとしても、自分で何かを始めるとしても役立つ、「誰かの困った、面倒、退屈、欲しい気持ちを見つけ、自分ができることで少し良くする力」を育てる。

起業セミナー本番で、子供たちが次のように感じられることを目指す。

- なんとなく怖い話ではなく、自分にも関係ある話だ。
- 価値って、人の状態が変わることなんだ。
- 仕事やお金は、誰かが助かることとつながっているんだ。
- アイデアは、すごいひらめきだけでなく、観察から作れるんだ。
- まず小さく試してみればいいんだ。

## 対象

- 中学2年生の男の子2人。
- 起業、事業、PMFなどの知識はほぼない。
- 机上の講義だけでは飽きやすい。
- ゲーム、部活、スマホ、勉強、テスト、友達、文化祭、お店などの例が入りやすい。

## 実施方法

- 全4日間。
- 1回30〜60分。
- 標準は45分。
- 家庭内対面またはZoomで実施可能。
- 親が話す時間より、子供が考える、話す、書く、選ぶ時間を多くする。

## 現在の作業フェーズ

現在は、Markdownで内容を固めるフェーズである。

この段階で行うこと:

- 要件定義を整える。
- 4日間の全体構成を整える。
- 成果物仕様を整える。
- 用語、トーン、禁止表現をそろえる。
- 実施に使うMarkdown教材を整える。

この段階で行わないこと:

- 画像生成。
- PDF生成。
- PPTX生成。
- PNG生成。
- HTML/CSS化。

これらは、人間が内容を確認した後に進める。

## 4日間の流れ

| Day | タイトル | 子供に残したい一言 | 主なワーク | 成果物 |
|---|---|---|---|---|
| Day1 | Value Detective | 仕事とは、誰かの状態を少し良くすること | 価値変換ワーク | 仕事とは、＿＿を＿＿に変えること |
| Day2 | Problem Hunter | 価値の種は、日常の不満、面倒、好き、欲しいにある | 価値の種ハンティング | 価値の種リスト3つ |
| Day3 | Idea & Want Signal Builder | 良いアイデアは、具体的な誰かが本当に欲しいものに近づけること | アイデア文＋小さな実験 | 私たちは、＿＿の＿＿を＿＿に変える |
| Day4 | Pitch Challenger | ピッチは、すごさ自慢ではなく価値の変化を伝えること | 3分ピッチ | 3分ピッチと振り返り |

## 各日の基本形

各日は、次の形を基本にする。

1. 目的。
2. 今日わかること。
3. 導入の問い。
4. 身近な例。
5. メインワーク。
6. 発表または共有。
7. まとめ。
8. 宿題または次回までの観察。

親の説明は短くする。
子供が考え、選び、書き、話す時間を多くする。

## 全体で使う共通フレーム

### 価値の変化フレーム

```text
誰が？
Before: 今どんな状態？
After: どう変わるとうれしい？
自分たちができること: 何をする？
```

### アイデア文フレーム

```text
私たちは、＿＿の＿＿を＿＿に変える。
```

例:

```text
私たちは、テスト前に何から始めればいいか迷う中2の、勉強の始めづらさを、最初の10分だけ決められる状態に変える。
```

### 小さな実験フレーム

```text
本当に欲しい人がいるか、まず＿＿で試す。
見たい反応は＿＿。
赤信号なら＿＿を変える。
```

## 親の役割

親は先生ではない。親は、問いを出す伴走者である。

親がやること:

- 短い問いを出す。
- 子供の答えをすぐ評価しない。
- 「誰がどう変わる？」に戻す。
- できるだけ子供の言葉を使う。
- 反応が悪い時も「失敗」ではなく「赤信号」として扱う。

親がやらないこと:

- 正解を先に言う。
- お金になりそうかだけで評価する。
- アイデアのすごさだけを褒める。
- 試す前に否定して止める。
- 起業を強制する。

## セミナー本番との接続

この4日間が終わると、子供たちは起業セミナーで出てくる言葉を次のように受け取れる。

| セミナーで出そうな言葉 | 事前ワークでの理解 |
|---|---|
| ビジネス | 誰かの困ったや欲しいを解決し続ける仕組み |
| 価値 | 相手の状態が少し良くなること |
| 顧客 | それを本当に欲しい具体的な人 |
| 課題 | Beforeの状態 |
| ソリューション | Afterへ変える方法 |
| PMF | 作ったものと欲しい人の気持ちがピタッとはまること |
| 検証 | 小さく試して反応を見ること |
| ピッチ | 価値の変化を短く伝えること |

## 成功条件

4日間の最後に、子供たちが次を自分の言葉で言えたら成功。

- 価値は、相手の変化。
- 仕事は、誰かの状態を少し良くすること。
- お金は、価値が届いたことへの対価。
- 価値の種は、困った、面倒、好き、欲しいの中にある。
- アイデアは、具体的な誰かに向ける。
- 完璧に作る前に、小さく試せばよい。
- 起業セミナーで、どんな人のどんな困りごとを解決しているかを見てみたい。

## 作業順序

1. 固定情報源を確認する。
2. 要件定義、全体構成、成果物仕様を整える。
3. 用語集、親の役割、進行原則を整える。
4. ワークシートをA4印刷前提で整える。
5. 親向けスライドを整える。
6. 子供向けスライドを整える。
7. インフォグラフィックprompt packetを整える。
8. 進行台本とカード教材を追加する。
9. 全体QAを行う。
10. 人間確認後に、PDF、PPTX、PNG、HTML/CSSへ進む。

## 人間確認ゲート

次に進む前に確認する。

- 子供2人に対して、例や問いが近いか。
- 1回45分でよいか。30分寄りにするか。
- 親向け説明が押し付けに見えないか。
- 子供向け資料が幼すぎないか。
- キャラクターとビジュアル方針を使ってよいか。
- 画像生成やPDF/PPTX化へ進んでよいか。
