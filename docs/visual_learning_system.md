# Visual Learning System: 発見ゲーム型の教材設計

## 基本方針

子供向け資料は、説明スライドではなく、発見ゲーム型の教材画面として設計する。

画面を見る子供が、次のどれかをしたくなることを目指す。

- 見る。
- 選ぶ。
- 書く。
- 話す。
- 試す。
- 直す。
- 発表する。

## 使ってよい画面タイプ

- Mission Board: 今日のミッションを示す。
- Detective Note: 観察したことを書く。
- Case File: 具体的な誰かの困りごとを整理する。
- Card Table: カードを並べて考える。
- Evidence Board: Before/Afterや証拠をつなぐ。
- Experiment Lab: 小さな実験を決める。
- Pitch Stage: 価値の変化を伝える。
- Reflection Log: 学びを振り返る。

## ページごとに決めること

各ページ、各スライド、各インフォグラフィックでは、次を必ず決める。

1. 画面タイプ。
2. 子供がする行動。
3. キャラクターがしている行動。
4. 見せたい価値の変化。
5. ゲーム要素が進める学習行動。
6. 合格基準。

## キャラクターの行動例

- 観察する。
- メモする。
- 指差す。
- カードを並べる。
- 付箋を書く。
- ラフスケッチする。
- 質問する。
- 試す。
- 反応を見る。
- 振り返る。
- 発表する。

## 不合格例

- 見出しと本文だけの普通の説明スライド。
- 丸アイコンだけのキャラクター。
- 顔だけで行動がないキャラクター。
- 「ミッション」と書いているだけで、子供の行動が変わらない。
- カードが飾りになっている。
- 価値が人の変化として見えない。

## 合格例

Day1の価値説明ページ:

- 画面タイプ: Evidence Board。
- 子供の行動: 「誰がどう変わったか」を探す。
- キャラクター行動: 価値探偵ユウがBefore/After写真を見比べている。
- 価値の変化: 「お腹がすいた → おいしく食べられた」。
- ゲーム要素: 変化カードを矢印でつなぐ。
- 合格基準: 子供が「価値は変化」と言える。

## 4日間での使い分け

| Day | 主な画面タイプ | 目的 |
|---|---|---|
| Day1 | Mission Board, Evidence Board, Card Table | 価値を変化として見る |
| Day2 | Detective Note, Case File | 価値の種を観察する |
| Day3 | Card Table, Experiment Lab | アイデアとPMFを小さく試す |
| Day4 | Pitch Stage, Reflection Log | 価値の変化を伝える |
