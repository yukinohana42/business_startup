# Skill: visual-storytelling-designer

## Purpose

抽象概念を、人物の変化、Before/After、図解、短い吹き出しで伝えるSkill。

## Use When

- 価値、お金、仕事、PMF、小さな実験を視覚化する時。
- スライドやインフォグラフィックの絵づくりを設計する時。
- キャラクターの行動を決める時。

## Required Inputs

- `DESIGN.md`
- `docs/visual_learning_system.md`
- 対象ページの学習目的

## Visual Principles

- 価値は人の変化として見せる。
- Beforeは笑いものにしない。
- Afterは大げさにしすぎない。
- 図解は関係や流れを見せる。
- キャラクターは顔だけでなく行動させる。

## Process

1. 抽象概念を1つ選ぶ。
2. それを中学生向けに言い換える。
3. Before/Afterを作る。
4. 誰が登場するかを決める。
5. キャラクターが何をしているかを決める。
6. 矢印、カード、ノートなどの図解要素を決める。
7. 吹き出しを1文以内にする。

## Acceptance Criteria

- 見るだけで「誰がどう変わるか」がわかる。
- 文章に頼りすぎていない。
- イラストが飾りではなく意味を持っている。
- 親が見ても安心できる。

## Must Not

- 既存作品風の絵柄を指定しない。
- キャラクターを丸アイコンだけにしない。
- 困っている人を笑わせない。
