# Skill: gamification-designer

## Purpose

学びを楽しくするために、ミッション、カード、進捗、Final Missionなどを設計するSkill。

## Use When

- 各日をミッション化する時。
- カードやバッジを設計する時。
- 子供が自分ごととして参加しやすくする時。

## Required Inputs

- `02_SOURCE_OF_TRUTH.md`
- `03_FOUR_DAY_CURRICULUM.md`
- `docs/visual_learning_system.md`

## Principles

- ゲーミフィケーションは目的ではなく足場。
- 勝ち負けより発見。
- 報酬より振り返り。
- バッジや進捗は能力評価ではなく、学びの行動の記録。
- 親はゲームマスターではない。

## Process

1. その日の学習目的を確認する。
2. ミッション名を作る。
3. 子供の行動を決める。
4. 使うカードやノートを決める。
5. Final Missionの穴埋め文を作る。
6. 親の問いを決める。
7. 禁止するゲーム要素を確認する。

## Output Format

```markdown
## DayX Mission
- Mission:
- Goal:
- Tools:
- Child Action:
- Final Mission:
- Parent Question:
- Do Not:
```

## Acceptance Criteria

- 学習目的が薄まっていない。
- 競争やランキングになっていない。
- 子供の発見が増える。
- Final Missionが学びを一文で固定している。

## Must Not

- 点数で競わせない。
- 稼げる仕事当てゲームにしない。
- 親をゲームマスターにしない。
