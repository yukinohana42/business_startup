# 02_SOURCE_OF_TRUTH

このファイルは、この4日間プロジェクトで絶対にぶらさない基本方針である。

## 目的

このプロジェクトは、中学2年生の子供たちに「起業家になれ」と教えるものではない。

会社に入るとしても、自分で何かを始めるとしても、誰かにとって価値あるものを見つけ、形にし、小さく試せる力を育てるための事前学習である。

## 最上位メッセージ

- 仕事とは、誰かへの価値提供である。
- お金は、価値提供に対する対価である。
- お金は、価値が届いたサインとして扱う。
- 価値は、相手の変化である。
- 価値は、言葉だけでなく「相手がどう変わったか」として見せる。
- 価値の種は、不満、面倒、好き、快感、欲しい気持ちの中にある。
- 顧客は「みんな」ではなく、具体的な誰かである。
- 良いアイデアとは、自分が作りたいものではなく、相手が本当に欲しいものである。
- PMFとは、作ったものと欲しい人の気持ちがピタッとはまることである。
- 起業は、完璧に考えるより、小さく試して、反応を見て、改善するものである。
- 赤信号は失敗ではなく、次に変えるサインである。

## 今回の言い換え辞書

| 難しい言葉 | 中学生向けの言い換え |
|---|---|
| 価値提供 | 困ったを助かったに変えること |
| 顧客 | それを本当に欲しい具体的な人 |
| PMF | 作ったものと欲しい人の気持ちがピタッとはまること |
| プロトタイプ | 完璧に作る前のお試し版 |
| 実験 | 小さく試して反応を見ること |
| 失敗 | 赤信号。次に変えるサイン |
| 事業 | 誰かの困りごとや欲しい気持ちを解決し続ける仕組み |
| お金 | 価値が届いたことへのありがとうが形になったもの |

## 見せ方の方針

子供が理解しやすいように、身近なストーリー、人物、場面で説明する。

使いやすい例:

- ゲーム
- 部活
- 勉強
- テスト
- 友達
- スマホ
- 親との会話
- 文化祭
- コンビニ
- ラーメン屋
- YouTube
- ゲーム実況

抽象論だけで終わらせない。必ず「誰が、どう困っていて、何が変わったか」に戻す。

## ゲーミフィケーションの扱い

ゲーミフィケーションは学びの足場であり、目的ではない。

- 勝ち負けより発見を重視する。
- 報酬より振り返りを重視する。
- バッジやポイントは評価ではなく、学びの行動の記録である。
- 親はゲームマスターではなく、問いを出す伴走者である。
- ミッション化は、子供が考えやすくなるために使う。
- 競争、ランキング、点数化は安易に使わない。

## 親の役割

親は、答えを教える人ではなく、問いを出すファシリテーターである。

親が先に正解を言うと、子供は自分の観察より「親が喜ぶ答え」を探しやすくなる。

親は次のように関わる。

- 「誰が一番困っていると思う？」
- 「今はどうやって解決している？」
- 「それが変わると何がうれしい？」
- 「小さく試すなら、何からできる？」
- 「反応が悪かったら、どこを変える？」

## 避ける表現

- 起業しないといけない。
- 会社員は危ない。
- サラリーマンは終わり。
- 将来困るから今やりなさい。
- 正解はこれ。
- それは無理。
- もっとすごいアイデアにしなさい。
- 稼げる人が偉い。
- 点数や順位で比べる。

## 使いたい表現

- 誰かにとって価値あることを見つける。
- 困ったを助かったに変える。
- 面倒をラクにする。
- 退屈を楽しいにする。
- もっとやりたいを形にする。
- まず小さく試す。
- 赤信号でも失敗ではなく、学び。
- 会社に入るとしても、自分で何かをやるとしても役立つ力。

## 人物イラストの扱い

人物イラストは抽象概念を具体化するために使う。

- 価値は、人の表情、行動、状態の変化として見せる。
- Before/Afterを使う。
- Beforeは相手を笑いものにしない。
- Afterは大げさな成功ではなく、少し良くなった状態にする。
- 完全オリジナルの同世代キャラクターを使う。
- 既存作品、既存キャラクター、特定作品の絵柄模倣はしない。
